# Projects

This directory contains active and completed projects. Each project should be a self-contained application or module with its own structure, dependencies, and documentation.

## Purpose

- **Production code** - Complete, tested, and deployable applications
- **Portfolio pieces** - Finished projects that showcase your work
- **Long-term initiatives** - Projects that span multiple phases or milestones
- **Reusable modules** - Libraries and utilities that can be shared or distributed

## Guidelines

Each project should have:

- A `README.md` with setup and usage instructions
- Clear folder structure and organization
- Appropriate test coverage
- Dependency management (`package.json`, etc.)
- Contributing guidelines if collaborative

## Structure

Organize projects by type or purpose:

```
projects/
├── web-app-template/
│   ├── README.md
│   ├── package.json
│   ├── src/
│   └── tests/
├── cli-tool/
│   ├── README.md
│   ├── package.json
│   └── bin/
├── api-server/
│   ├── README.md
│   ├── package.json
│   └── src/
└── utilities/
    ├── logger-lib/
    └── validation-lib/
```

## Project Checklist

When starting a new project, ensure:

- [ ] Clear README with purpose and usage
- [ ] Defined scope and goals
- [ ] Appropriate scaffolding and tooling
- [ ] Version control initialized
- [ ] Dependencies documented
