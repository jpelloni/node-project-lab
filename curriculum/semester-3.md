# Semester 3 – Concurrency, Performance, and Scalability

## Mission

Writing software is only the beginning. Building software that performs reliably under load is a different challenge.

This semester explores how Node.js manages concurrency, parallelism, memory, and system resources. We'll investigate performance bottlenecks, learn to profile applications, and understand when to introduce worker threads, child processes, and distributed workloads.

The objective is to become comfortable reasoning about performance rather than relying on assumptions.

## Learning Objectives

* Understand Worker Threads and Child Processes.
* Explore process isolation and parallel execution.
* Learn caching strategies.
* Understand queues and background processing.
* Benchmark and profile Node applications.
* Investigate memory usage and garbage collection.
* Apply performance improvements based on measured data.

## Capstone

Build a scalable processing service capable of handling large workloads while maintaining responsiveness and efficient resource utilization.
