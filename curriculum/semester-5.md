# Semester 5 – Observability and Diagnostics

## Mission

Production software cannot be improved if it cannot be understood.

This semester focuses on making applications observable. We'll explore logging, metrics, tracing, profiling, diagnostics, and performance analysis to understand what applications are doing while they are running.

The emphasis is on building tools that explain behavior rather than simply reporting failures.

## Learning Objectives

* Build structured logging systems.
* Collect application metrics.
* Explore distributed tracing concepts.
* Learn CPU and memory profiling.
* Analyze heap snapshots.
* Understand AsyncLocalStorage.
* Build diagnostic tooling for Node.js applications.

## Capstone

Build a lightweight observability toolkit capable of collecting runtime diagnostics, performance metrics, and execution traces.
