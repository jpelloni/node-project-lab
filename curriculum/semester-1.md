# Semester 1 – The Node Runtime

## Mission

Every Node.js developer knows how to write JavaScript. Fewer understand what happens between running `node app.js` and the first line of application code executing.

This semester focuses on the foundations of the Node.js runtime. Rather than treating Node as a black box, we'll explore how JavaScript executes, how asynchronous operations are scheduled, how files are read, and how Node's core APIs are built.

Each lesson introduces a single concept, reinforces it through experimentation, and concludes with a focused project designed to answer one engineering question.

By the end of this semester, I should understand not only how to use Node.js, but why it behaves the way it does.

## Learning Objectives

* Understand the architecture of the Node.js runtime.
* Learn how JavaScript execution differs inside Node.
* Understand the Event Loop and libuv.
* Explore Buffers, Streams, Events, Timers, and the File System.
* Read portions of the Node.js source code.
* Build small utilities without relying on external libraries.
* Develop confidence reasoning about runtime behavior and performance.

## Capstone

Build a miniature runtime monitor that demonstrates the concepts learned throughout the semester.
